# ADR — fix-log-timestamp

**작성일:** 2026-04-16
**작성자:** planner (초안) / reviewer (검토 후 보완)

## 결정 1: log.md 시간 획득 방법으로 `date +%H:%M` bash 명령 채택

**상태:** 제안

**맥락**
에이전트들이 log.md에 시간을 기록할 때 시스템 프롬프트의 `currentDate` 값(날짜만 포함)을 해석하여 시간을 추정하고 있다. 이 값에는 시간 정보가 없으므로 UTC 자정(00:00)으로 해석되며, KST 변환 시 항상 09:00이 된다. 실제 작업 시각과 무관한 시간이 기록되어 로그의 신뢰성이 떨어진다.

**결정**
각 에이전트의 Document_Responsibility 섹션에 log.md 시간 기록 시 반드시 `date +%H:%M` bash 명령을 실행하여 실제 시스템 시간을 획득하라는 지시를 추가한다. docs/FORMATS.md에도 동일한 규칙을 명시한다.

**고려한 대안**
- 대안 A: 시스템 프롬프트에 시간 정보 포함 — 시스템 프롬프트는 정적이므로 세션 시작 시점의 시간만 반영 가능하며, 장시간 세션에서는 여전히 부정확하므로 거부
- 대안 B: JavaScript Date() 사용 — 에이전트가 bash 환경에서 동작하므로 bash 명령이 더 자연스럽고 직접적이므로 거부

**결과**
모든 에이전트가 log.md 기록 시 실제 시스템 시간을 사용하게 되어 로그의 정확성과 신뢰성이 향상된다. 에이전트 .md 파일 7개(에이전트 6개 + FORMATS.md 1개) 수정이 필요하다.
